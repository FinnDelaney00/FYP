# Layers Folder Guide

This folder contains the shared Python dependency layer used by the ML-related Lambdas.

## What Is Here

| Path | Purpose |
| --- | --- |
| `ml/requirements.txt` | Source dependency list for the ML/anomaly shared layer |
| `ml_dependencies.zip` | Packaged Lambda layer artifact generated by Terraform |

## Why The Layer Exists

The forecasting and anomaly Lambdas both depend on heavier scientific Python packages:

- `scikit-learn`
- `pandas`
- `numpy`
- `joblib`

Keeping these packages in a shared layer avoids duplicating them across multiple Lambda zip files.

## How The Layer Is Built

Terraform uses:

- `scripts/package_python_layer.py`
- a `terraform_data` resource with `local-exec`
- `data "archive_file"` to zip the generated layer directory

The generated build directory is:

- `build/ml_layer/python/...`

The final zipped artifact is:

- `layers/ml_dependencies.zip`

## Compatibility Notes

- The layer is built for Lambda Python `3.11`
- The packaging flow targets `manylinux2014_x86_64`
- The ML and anomaly Lambdas both use `x86_64`

If you change package versions in `ml/requirements.txt`, Terraform will rebuild the layer automatically because the requirements hash is part of the replacement trigger.

## When You Edit This Folder

You typically touch this folder when:

- upgrading `numpy`, `pandas`, or `scikit-learn`
- fixing Lambda layer compatibility issues
- shrinking or rebuilding the ML dependency bundle

If you are adjusting the build behavior itself, see [../scripts/README-scripts.md](../scripts/README-scripts.md).
